<task type="auto">
  <name>Task 4: Implement AccountStep input handling with Tab/arrow navigation</name>
  <files>src/wizard.rs</files>
  <action>
    Implement `handle_input()` for AccountStep:

    Handle these key patterns:
    - **Up/Tab**: `focus_field = focus_field.saturating_sub(1)` (move to previous field)
    - **Down/BackTab**: `focus_field = min(focus_field + 1, 4)` (move to next field)
    - **Enter**: 
      - If field has valid content, auto-advance to next field
      - If on last field (git_email) and valid, mark step complete and go_next()
      - If field is invalid, set validation_error and show summary
    - **Char(c)**: Append to current field's buffer
    - **Backspace**: Remove last char from current field's buffer
    - **Ctrl+U**: Clear current field buffer

    On Enter with invalid field:
    - Validate current field
    - Set validation_error = error message if invalid
    - Do NOT advance

    On navigation (Up/Down/Tab) with invalid field:
    - Clear validation_error (user is fixing it)
  </action>
  <verify>`cargo check --lib 2>&1 | grep -c "error"` - 0 errors, input handling compiles</verify>
  <done>AccountStep handles Tab/arrow navigation and Enter to advance</done>
</task>
