  <name>Task 3: Implement TimezoneStep input handling with type-to-filter</name>
  <files>src/wizard.rs</files>
  <action>
    Implement handle_input() for TimezoneStep:

    Key handling:
    - **Up/Down**: Navigate within filtered list (respects current filter)
    - **Enter**: Select timezone, mark step complete, go_next()
    - **Char(c)**: Append to filter string, apply_filter(), reset selected to 0
    - **Backspace**: Remove from filter, apply_filter()
    - **Ctrl+W**: Clear filter (Ctrl+Backspace)
    - **Escape**: If filter has content, clear it; otherwise go_back()

    Type-to-filter behavior per 02-CONTEXT.md:
    - "Type-to-filter starts immediately when user types any character"
    - Filter is case-insensitive substring match
    - Navigation keys exit filter mode and navigate list

    When filter changes: selected = 0 (jump to top of filtered results).
  </action>
  <verify>`cargo check --lib 2>&1 | grep -c "error"` - 0 errors for input handling</verify>
